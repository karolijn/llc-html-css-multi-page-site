##Submitting a form Forms are used to capture information from a user. The information processing and form submission usually requires the use of a server-side language such as PHP, Ruby or Python. But there are different services available to help do the processing for us without writing any of these languages. Let's first look at what each of these form controls do. * `input` tag are used to allow the user to type in their information and is used with a `type` attribute to determine the kind of information it is. * The `label` tags are used to label the input or else the user would not know what information to enter. * The `name` attribute is used to label the fields when sending the form data to the server. * The `for` and `id` attributes associate the label to the input and should have matching values. A form can contain any HTML markup (paragraphs, lists, etc.) in addition to form specific HTML tags. [Each part of a form is considered a paragraph](http://dev.w3.org/html5/spec-LC/forms.html#forms) and can be separated using `<p>` elements. **Further reading: ** [MDN forms](https://developer.mozilla.org/en-US/docs/Web/Guide/HTML/Forms)
##3rd Party services There are many services that help with form submission. Let's look at [FORMS by Brace](http://forms.brace.io/).Through this service, the information will be forwarded to your email. Simple! Remember, all forms begin with the `form` element and requires two attributes, `method` (defines how the form will communicate with the web server) and `action` (provides the file path to where the form script is processed after the form is submitted. Since we're using a 3rd party service, the `action` value is *their* url *and* the email you want the form to be submitted to. ```xml <form method="POST" action="http://forms.brace.io/myemail@hello.com"> ... </form> ```
##3rd Party services Brace.io requires that you verify your email first. The verification email will be sent on the first form submit so make sure you test your form first! Also note, this service only works on a *server* so you can't test it *locally*, meaning, just from your computer. You'll need to upload it to a server or run a local server to test it. See [getting your site online](../index.html#slide89) or [how to run a local server](http://mac.appstorm.net/reviews/internet-reviews/running-a-local-server-with-mamp/). See further documentation here: [http://forms.brace.io/](http://forms.brace.io/) Another services to check is [Wufoo](http://www.wufoo.com/). You can even use Google Docs to create a form that will add the form submissions into a spreadsheet.
